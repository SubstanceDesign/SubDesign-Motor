Default text inputs

To create default text input add .text-input class to <input type=text>, <textarea> or <select> tag.

Code

<input type="text" class="text-input" placeholder="Default text input">

<textarea class="text-input" placeholder="Default text input" rows="4"></textarea>

<select class="text-input" placeholder="Default text input">
  <option disabled selected>Default selectbox</option>
  <option>Option 1</option>
  <option>Option 2</option>
  <option>Option 3</option>
</select>

Validation states

For validation states use .error, .warning and .success classes.

Code

<input type="text" class="text-input error" placeholder="Error">
<input type="text" class="text-input warning" placeholder="Warning">
<input type="text" class="text-input success" placeholder="Success">

Demo

Also you can use state classes for a parent of input.

<div class="input-wrapper error">
  <input type="text" class="text-input" placeholder="Error">
</div>

Disabled state

For disabled state add ordinary disabled attribute. Also you can use .disabled for specific purposes.

Code

<input type="text" class="text-input" placeholder="Default text input" disabled>

Demo

Input sizes

Text inputs have three size modes excluding default. To change size add corresponding modifier .text-input--xs, .text-input--sm, .text-input--lg.

Code

<input type="text" class="text-input text-input--xs" placeholder="Default text input">
<input type="text" class="text-input text-input--sm" placeholder="Default text input">
<input type="text" class="text-input text-input--lg" placeholder="Default text input">

Semantic classes / Mixins

If you are a fan of semantic and clean HTML code, you may like to use your own class names for structure blocks. We designed mixins that can apply framework styles to your own classes.

Mixins list

Mixin Description Variables 
add_text-input_styles()
 Add necessary for any text input settings and style resets.
You shouldn't use it with create_text-input mixins. It is included already		 - 
create_text-input($background, $color, $border, $border-focus)
 Create default text input $background: color of input background. Required!

$color: color of input text. Required!

$border: color of input border. Required!

$border-focus: color of focused input border. Required!

Examples:

SCSS

.custom-input-name {
  @include create_text-input(#000, #fff, #ccc, #ddd); // apply default text input styles
}

HTML

<input class="custom-input-name" type="text">
Also you can include common input mixins. For example size mixin.

SCSS

.custom-input-name {
  @include create_text-input(#000, #fff, #ccc, #ddd); // apply default text input styles
  @include add_input_size(60px, 20px, 16px, 3px, 4px);
}

Setting text inputs config

The framework has common input settings and individual text input settings. Settings are in _source/config.scss file.

Settings list

Name Description Values
$text-input_border-width Border width of default text-input px
$text-input_default Enable default text-inputs true / false

Autoresize textarea

You can use autoresize textarea feature implemented with javascript. Just add data-textarea-autoresize attribute to textarea.

Code

<textarea class="text-input" placeholder="Autoresize" rows="4" 
  data-textarea-autoresize 
  data-textarea-autoresize-max-height='300'></textarea>

Demo

Also you can call the script directly from js.

JS

motor( document.getElementById('my-textarea') ).TextareaAutoresize({
  maxHeight: 300
});

Demo

Settings list

Name Description Values
maxHeight

data-textarea-autoresize-max-height		 Height limit for textarea autoresize number
updateOnResize

data-textarea-autoresize-update-on-resize		 Update textarea height on window resize true / false

Methods list

Name Description
updateHeight() Check textarea overflow and update its height if needed

EXAMPLE

// Initialize plugin
var textareaAutoresized = motor( document.getElementById('my-textarea') ).TextareaAutoresize();

// Increase textarea value to be anormous long
document.getElementById('my-textarea').value = '...Very long value...';

// Update textarea height to fit its new scrollable content
textareaAutoresized.updateHeight();

The plugin work is accompanied by custom event calls. To listen to framework custom events use motor(element).addCustomEventListener( eventName, eventHandler ) method.
For modern browsers you can use default element.addEventListener method.

Events list

Name Description
textarea-autoresize-init Fires on textarea plugin initialized
textarea-autoresize-resize Fires on textarea resize

EXAMPLE

var textarea = document.getElementById('my-textarea');

// Add init event listener 
motor(textarea).addCustomEventListener( 'textarea-autoresize-init', function() {
  console.log('Textarea autoresize was inited');
} );

// Add resize event listener 
motor(textarea).addCustomEventListener( 'textarea-autoresize-resize', function() {
  console.log('Textarea autoresize was resized');
} );

// Initialize plugin
motor(textarea).TextareaAutoresize();

Also you can call motor methods and plugins with selector string, like in example below. But note that motor.js use native js querySelectorAll method, which can accept not all the CSS3 selectors for IE8.

motor('#my-textarea').TextareaAutoresize({
  maxHeight: 300
});
