Responsive columns

The grid is responsive. You can control each column width with viewort prefixed classes. By default you have six viewport variants for the all grid features:

  • 'mc_' (initial),
  • 'xs_' (from 480px),
  • 'sm_' (from 750px),
  • 'md_' (from 970px),
  • 'lg_' (from 1120px),
  • 'xl_' (from 1280px).

That gives you possibility to apply column's or row's class for particular screen size.

The grid is mobile-first. This means that first of all styles for the smallest screens are applied. And styles for the bigger screens will inherit the properties and override them or assign unique.

Code

<div class="row">
  <div class="sm_col-4">.sm_col-4</div>
  <div class="sm_col-4">.sm_col-4</div>
  <div class="sm_col-4">.sm_col-4</div>
</div>

Nested columns

You can put rows inside columns and create complicated grid structure.

Code

<div class="row">
  <div class="sm_col-6">
    <div class="row">
      <div class="sm_col-6">.sm_col-6</div>
      <div class="sm_col-6">.sm_col-6</div>
    </div>
  </div>
  <div class="sm_col-3">.sm_col-3</div>
  <div class="sm_col-3">.sm_col-3</div>
</div>

Columns with offset

A column can be moved to the right on necessary amount of columns with .col--offset-* classes.

Code

  <div class="sm_col-6">.sm_col-6</div>
  <div class="sm_col-3 sm_col--offset-3">.sm_col-3.sm_col--offset-3</div>
</div>

Change columns order

Columns order can be changed with .col--right-* and .col--left-* classes. In order to swap columns you need to move them toward each other by the width of one another. Also with this feature you can mix columns the way you like.
Class .col--reset-order will reset order to default.

Code

<div class="row">
  <div class="col-8 sm_col--right-4 lg_col--reset-order">.col-8.sm_col--right-4.lg_col--reset-order <br><br>Column will be moved to the right on screens from 768px <br>And it will returns to the left on screens from 1200px</div>
  <div class="col-4 sm_col--left-8 lg_col--reset-order">.col-4.sm_col--left-8.lg_col--reset-order <br><br>Column will be moved to the left on screens from 768px <br>And it will returns to the right on screens from 1200px</div>
</div>

Custom class for a column or a row / Mixins

If you are a fan of semantic and clean HTML code, you may like to use your own class names for structure blocks. We designed mixins that can apply grid styles to your own classes.

Mixins list

  // any content
}

Use this mixin to responsively apply row features to your classes.		 $breakpoint: one of the viewport shortnames (mc, xs, sm, md, lg, xl). Required!

 Apply .grid styles to an element -

create_row--vertical-gutter($gutter-width);

 Apply center alignment to columns of a target row $include-flexbox-mixin: true / false. Optional. Set false to not iclude common alignment features styles again.

Examples:

SCSS

.custom-grid-name {
  @include create_grid(); // extend .grid styles
}

.custom-row-name {
  @include create_row(); // extend .row styles
  @include viewport(md) {
    @include create_row--gutter(120px); // apply 120px gutter to the row on medium viewport (from 992px)
  }
}

.custom-col-name {
  @include create_col(4, sm); // extend .sm_col-4 styles
  @include create_col(3, md, false); // extend .md_col-3 styles
  @include create_col--offset(4, sm); // extend .sm_col--offset-4 styles
}

HTML

<div class="custom-grid-name">
  <div class="custom-row-name">
    <div class="custom-col-name"></div>
    <div class="custom-col-name"></div>
    <div class="custom-col-name"></div>
  </div>
</div>

Also you can use SASS directive @extend. Just extend the grid classes you need to reach the same effect.

SCSS

.custom-class-1 {
  @extend .col-8;
  @extend .sm_col--right-4;
  @extend .lg_col--reset-order;
}

.custom-class-2 {
  @extend .col-4;
  @extend .sm_col--left-8;
  @extend .lg_col--reset-order;
}

HTML

<div class="row">
  <div class="custom-class-1">.col-8.sm_col--right-4.lg_col--reset-order</div>
  <div class="custom-class-2">.col-4.sm_col--left-8.lg_col--reset-order</div>
</div>

Setting grid config

The framework has grid config on global and local levels. Global settings are in _source/config.scss file. Local settings are in _source/core/grid/scss/config.scss file. Local settings can be used in standalone mode, when you like to use component without whole framework. Or them are useful in case of overriding global settings.

You can set number of columns in grid, change gutters and grid size on different viewports.

Config let you disable features that you don't need. It is a good practice to reduce css file size.

Settings list

Name Description Values

By default array of viewports equal to value in global framework config
grid_feature_responsive Enable responsive grid true / false
grid_feature_offset Enable offset for columns true / false
grid_feature_columns-order Enable columns order feature true / false
grid_feature_row--collapsed Enable collapse gutters in row feature true / false
grid_feature_different-gutters Enable different gutters in row feature true / false
grid_feature_vertical-gutters Enable vertical gutters feature true / false
grid_feature_horizontal-alignment Enable horizontal alignment feature true / false
grid_feature_vertical-alignment Enable vertical alignment feature true / false
grid_cols Set number of columns in grid Any integer
grid_gutter Set default gutter size px
grid_gutters Set different gutters array Key - any string, value - px
viewport_content-sizes Set grid sizes for different viewports Keys - reserved (do not change them), value - px or '100%'
