Default switch

To use motor default switch create <label> tag with .switch class. Put input[type=checkbox] or input[type=radio] inside it, and create span.switch_lever tag next to input. You can place label text for on and off states after that in span.switch_on and span.switch_off.

Code

<label class="switch">
  <input type="checkbox" name="switch-1">
  <span class="switch_lever"></span>
  <span class="switch_on">on</span>
  <span class="switch_off">off</span>
</label>

<label class="switch">
  <input type="radio" name="switch-radio" checked>
  <span class="switch_lever"></span>
</label>
<label class="switch">
  <input type="radio" name="switch-radio">
  <span class="switch_lever"></span>
</label>
<label class="switch">
  <input type="radio" name="switch-radio">
  <span class="switch_lever"></span>
</label>

Switch styles

Besides default motor switch style you can use doted style with .switch-doted and ios style with .switch-ios classes.

Code

<label class="switch-doted">
  <input type="checkbox" name="switch-2">
  <span class="switch_lever"></span>
  <span class="switch_on">on</span>
  <span class="switch_off">off</span>
</label>
<label class="switch-ios">
  <input type="checkbox" name="switch-3">
  <span class="switch_lever"></span>
  <span class="switch_on">on</span>
  <span class="switch_off">off</span>
</label><

Disabled state

For disabled state add ordinary disabled attribute to input and .disabled class to it for old browsers support.

Code

<label class="switch">
  <input type="checkbox" name="switch-4" disabled>
  <span class="switch_lever"></span>
  <span class="switch_on">on</span>
  <span class="switch_off">off</span>
</label>

Inline modificator

For the inline-block behavior use .switch--inline class.

Code

<label class="switch switch--inline">
  <input type="radio" name="switch-8" checked>
  <span class="switch_lever"></span>
</label>
<label class="switch switch--inline">
  <input type="radio" name="switch-8">
  <span class="switch_lever"></span>
</label>
<label class="switch switch--inline">
  <input type="radio" name="switch-8">
  <span class="switch_lever"></span>
</label>

Semantic classes / Mixins

If you are a fan of semantic and clean HTML code, you may like to use your own class names for structure blocks. We designed mixins that can apply framework styles to your own classes.

Mixins list

Mixin Description Variables 
create_switch(
  $width, 
  $height, 
  $border-width, 
  $on-color, 
  $off-color, 
  $tumbler-color, 
  $border-color, 
  $dot: true
)
 Create default or doted switch. $width: switch width in px. Required!

$height: switch height in px. Required!

$border-width: switch border-width in px. Required!

$on-color: on state color. Required!

$off-color: off state color. Required!

$tumbler-color: tumbler color. Required!

$border-color: switch border color. Required!

$dot: true / false. Optional. By default: false.		 
create_switch-ios(
  $width, 
  $height, 
  $border-width, 
  $on-color, 
  $off-color, 
  $border-color
)
 Create ios styled switch. $width: switch width in px. Required!

$height: switch height in px. Required!

$border-width: switch border-width in px. Required!

$on-color: on state color. Required!

$off-color: off state color. Required!

$border-color: switch border color. Required!

Examples:

SCSS

.custom-switch-name {
  @include create_switch-ios(60px, 40px, 1px, #000, #fff, #ccc);
}

HTML

<label class="custom-switch-name">
  <input type="checkbox" name="switch-name">
  <span class="switch_lever"></span>
  <span class="switch_on">on</span>
  <span class="switch_off">off</span>
</label>

Setting switch config

The framework has common input settings and individual checkbox and radio settings. Settings are in _source/config.scss file.

Settings list

Name Description Values
$switch_width Switch width px
$switch_height Switch height px
$switch_border-width Switch border-width px
$switch_border-color Switch border-color color
$switch_default Enable default switch true / false
$switch_doted Enable doted switch true / false
$switch_ios Enable ios switch true / false
